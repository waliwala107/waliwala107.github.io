0x02 什么是本地提权？

讲具体技术前，大家先了解下：什么是本地提权？

权限提升是利用操作系统或软件应用程序中的错误、设计缺陷或配置疏忽来获得对通常受应用程序或用户保护的资源的提升访问权限的行为。 ——维基百科

“本地提权”可使黑客实施网络攻击时获得系统最高权限，从而取得对网站服务器的控制权。 ——百度百科

通俗来讲：本地提权就是说将一个本来非常低权限、受限制的用户，提升到系统超级用户的权限。

0x03 为什么要本地提权？

当成功通过80或者443端口通过web服务渗透时，获取的shell用户通常是www-data 。无法执行root 权限下的一下命令或者读取/root 下的重要文件。这个时候就需要本地提权，在root 权限下，还可以通过msfvenom生成其他后门文件或者一些隐藏后门。添加用户，开启其他端口等操作，达到权限持续控制。

0x0404 本地提权方式

本地提权整体分为3部分：Credential Access, Exploit, Misconfiguration。

Credential Access

Reused Passwords

Password reuse is a person’s tendency to use the same password across different online services.

密码重用，顾名思义就是不同服务使用同一个账号和密码。实际测试过程中也遇到过，web的登陆账号密码和root用户的ssh密码一样，攻击者直接可以利用su命令来提权为root用户。

Credentials from configuration files

从服务器的配置文件中获取到密码信息，接着利用密码重放来登陆root的ssh服务。通常时候，拿到web的反弹shell之后，习惯性去翻一下web的配置文件，里面经常会明文保存数据库的用户信息，数据库的连接密码有时候就是root的ssh登陆密码。

Credentials from local database

拿到后台数据库登录信息后，在受害机上尝试登陆数据库，在数据库中获取敏感信息，其中可能就包含root用户的密码。

Credentials from bash history

bash history文件是记录用户命令的文件，通常情况下拿到反弹shell之后可以去检查用户的命令历史，其中也许存在明文密码。例如：数据库登录信息、远程桌面连接信息等等。

SSH keys

已知用户家目录的.ssh隐藏文件夹中保存有ssh公私钥信息，如果文件权限设置不当，攻击者可以读取私钥信息，那么攻击者可以将私钥在本地复制一份，然后对私钥文件赋予600权限，最后通过ssh -i来实现登陆root用户。不过前提是root用户的.ssh文件中有authorized_keys，并且文件中写入了对应的公钥。

Sudo Access

Sudo 是一个 Linux 实用程序，它允许用户以其他用户的权限运行命令，当不提供任何参数时，它将以 root 用户身份执行命令。
如果 sudo 配置不正确，这可能允许攻击者将他们的权限提升到 root。

通常情况下拿到反弹shell之后，习惯性的执行sudo -l 来检查当前用户允许通过sudo执行哪些命令。有一个方便的在线工具可以帮助我们来快速利用sudo实现提权。

工具链接：https://gtfobins.github.io/

不过尽信书不如无书，工具只是一个辅助，经过实际测试有些命令该网站就没有收录，所以最终还是需要使用者来判断。例如：/bin/crm、kubectl cp等等也是可以通过sudo实现提权。

Exploit

Services Running on Localhost

利用当前运行的服务来实现本地提权，例如著名的UDF提权，利用的就是mysql服务。

UDF提权的知识参考：https://hakin9.org/privilege-escalation-with-mysql-user-defined-functions/

Kernel version

内核提权是一种比较常见的本地提权方式，实际过程中，运维很少会及时更新内核或操作系统，尤其是一些小公司。著名的内核提权漏洞有脏牛(CVE-2016-5195)。

脏牛漏洞的知识参考：https://dirtycow.ninja/

Binary file version

系统本地运行的二进制文件存在本地提权漏洞，最近比较著名的就是Polkit Privilege Escalation（CVE-2021-4034）

Polkit漏洞的知识参考：https://blog.qualys.com/vulnerabilities-threat-research/2022/01/25/pwnkit-local-privilege-escalation-vulnerability-discovered-in-polkits-pkexec-cve-2021-4034

Misconfiguration

Cron Job

定时任务提权一般要满足两个条件，第一任务是root用户执行，第二执行的内容普通用户有修改的权限。

正常情况下/var/spool/cron/crontabs/root文件普通用户是没有权限查看的，这里需要用到一个工具pspy。这款工具可以监控当前运行的服务，可以看看是否有root用户定时执行的服务。

工具链接：https://github.com/DominicBreuker/pspy

有时候会遇到定时执行的脚本文件普通用户无权修改，但是发现脚本中会导入某些python库，这些库文件是普通用户可以修改的，那么通过修改库文件也可以造成本地提权。

SUID/SGID files

SUID：设置用户ID是一种权限，允许用户以指定用户的权限执行文件。具有 suid 权限的文件以更高的权限运行。假设我们以非 root 用户身份访问目标系统，并且发现启用了 suid 位的二进制文件，那么这些文件/程序/命令可以以 root 权限运行。

可以知道哪些文件含有suid权限，接着通过https://gtfobins.github.io/工具来获取利用方式。

Sensitive files writeable

敏感信息可写，指的是普通用户可以修改例如：/etc/passwd, /etc/shadow, /etc/sudoers, configuration files。

假如passwd或者shadow文件可写，普通用户就可以在文件中添加一个uid=0的超级用户，从而实现提权。

假如sudoers文件可写，那么用户可以将自己添加到sudoers中，设置为无需密码可以执行任意命令，从而实现提权。

Sensitive files readable

敏感信息可读，指的是普通用户可以读取例如：/etc/shadow，/root/.ssh/id_rsa。

普通用户可以读取shadow文件中的hash值，利用工具进行爆破来获取root用户的密码。
